# Keryx Phase 3 - Reliable Family Communication

Phase 2 (FCM, sleeping architecture, push notifications, lightweight sockets) is complete.

Do NOT redesign the project. Build on the existing architecture.

The goal is to make Keryx feel like a tiny private family communication system while keeping RAM, CPU and battery usage extremely low.

The app is ONLY for two people:
- Mother
- Brother

No authentication system.
No contacts.
No groups.
No media.
No video calls.

---------------------------------------
1. SQLite Persistence
---------------------------------------

Replace all in-memory storage with SQLite.

Store:
- Messages
- Read status
- Delivered status
- Call logs
- Missed calls
- Last seen timestamps
- Emergency events

The database should remain lightweight.

---------------------------------------
2. Message Status
---------------------------------------

Implement:

✓ Sent

✓✓ Delivered

✓✓ Read

Exactly like modern messaging apps.

---------------------------------------
3. Presence
---------------------------------------

Show:

- Online
- In Call
- Sleeping (Push Active)
- Last Seen

Last seen should update automatically whenever the user disconnects.

---------------------------------------
4. Typing Indicator
---------------------------------------

Show:

"Mom is typing..."

or

"Brother is typing..."

Hide it immediately when typing stops.

---------------------------------------
5. Call Logs
---------------------------------------

Create call history.

Store:

- Incoming
- Outgoing
- Missed
- Duration
- Timestamp

If a call is not answered, automatically create a Missed Call entry inside chat.

---------------------------------------

Display:

Incoming

Time

Keep UI minimal.

---------------------------------------

---------------------------------------
8. Admin Notifications
---------------------------------------

One additional trusted user (me) receives important notifications.

Examples:

- Mom tried calling Brother but nobody answered.

- Brother missed an Emergency Call.

- More than 3 failed calls happened.

- Someone attempted to join using an invalid Family Key.

- Server restarted.

These notifications help me contact my family if something goes wrong.

---------------------------------------
9. Anti Spam
---------------------------------------

Prevent accidental call spam.

Maximum:

5 call attempts per minute.

If exceeded:

"Too many calls.
Please wait one minute."

Do not disconnect the user.

Simply block additional call attempts temporarily.

---------------------------------------
10. Auto Cleanup
---------------------------------------

Automatically remove:

Old logs

Old notifications

Old temporary data

after 30 days.

Do NOT remove chat history.

Keep SQLite compact.

---------------------------------------
11. Emergency Mode
---------------------------------------

Add a new button:

🚨 Emergency

When pressed:

- Send an Emergency message.

- Send an Emergency Push Notification.

- Display "Emergency Call" instead of normal call.

If unanswered:

Create a Missed Emergency Call log.

Also notify the Admin user.

---------------------------------------
12. Keep Lightweight
---------------------------------------

Preserve the sleeping architecture.

When idle:

- Disconnect WebSocket after timeout.
- No unnecessary polling.
- No background CPU usage.
- No background microphone.
- No unnecessary timers.

Use Firebase Cloud Messaging to wake devices.

Only reconnect when:

- User opens the app.
- Push notification is tapped.
- User starts a call.
- User sends a message.

The goal is extremely low RAM, battery and CPU usage, making Keryx suitable for low-powered Render servers and Android devices with aggressive battery management.

---------------------------------------
7. Private Family Lock
---------------------------------------

The application must NEVER allow random users to connect.

There are no accounts, usernames or passwords.

Instead, before entering the app, the user is asked for a single Family Key.

Example:

Family Key:
ramatulasi

The entered key must NEVER be stored in plain text.

Server Requirements:

- Store only a hashed version of the Family Key using bcrypt.
- Keep the hash inside an environment variable:

FAMILY_KEY_HASH=<bcrypt hash>

- When a user enters the Family Key:
    - Compare it against the bcrypt hash.
    - Never expose the real key.
    - Never send the key back to clients.
    - Never hardcode the key in source code.

If the key is correct:
- Allow access.

If incorrect:
- Reject immediately.
- Log the failed attempt.
- Increase failed-attempt counter.

Security:

- Maximum 5 incorrect attempts per hour from the same IP.
- After that:
    "Too many incorrect attempts. Please try again later."

Admin Notification:

If more than 3 invalid Family Key attempts occur within one hour:

Send an Admin Notification:

⚠️ Someone attempted to access Keryx using an invalid Family Key.

Include:

- Time
- Approximate IP address (if available)
- Attempt count

Do not expose any sensitive information to the user.

The Family Key can be changed later by replacing the bcrypt hash in the server environment variables without modifying application code.
